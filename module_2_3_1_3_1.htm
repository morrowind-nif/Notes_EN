Special Parent Nodes

Системные объекты с уникальными IDE.
AvoidNode
—  нода отвечающая за работу АИ!
Поверхность содержащая эту ноду запрещает ботам движение по ней.
 
NiBSAnimationNode
— анимационная нода.
Сообщает движку, что содержимое должно проигрывать постоянную анимацию!
Важно! Помимо названия должен быть еще правильный флаг.
Иначе анимация работать не будет.
См. раздел за подробностями.
 
NiBsAnimationManager
— антипод анимационной ноды.
Если в такой контейнер поместить объекты, их анимации будут заморожены.
Однако, использование этого объекта не предполагается в обычных ниф файлах.
Т.е. этот контейнер используется движком для управления анимациями объектов в сцене.
 
NiBillboardNode
— отвечает за выравнивание содержимого этой ноды по отношению к камере. 
Позволяет получить ряд интересных эффектов. Например ореолы у светильников.
 
NiBSPArrayController
— это не нода, это контроллер анимации частиц обладающий эффектом Именной Ноды.
Отчего и попал в этот раздел.
Т.е. имя этого контроллера оказывает решающий эффект на поведение частиц.
 
Отвечает за создание копий системы частиц по цепочке объектов.
Т.е. если есть некоторая иерархия (цепочка) объектов, то все объекты начиная от второго и до предпоследнего, получат копию исходной системы частиц!
Также, позволяет использовать поверхности шейпов в качестве эмиттеров.
Т.е. создает не только клоны частиц по иерархии, но и может включать использование вертексов модели в качестве эмиттеров!
 
NiBSPNode
— отвечает за фильтрацию объектов с особыми настройками работы альфа свойств.
Создает плоскость при пересечении которой, один из объектов будет становится невидимым.
Довольно специфический объект.
 
NiBSParticleNode
— нода частиц.
Является аналогом niBSAnimationNode, только для систем частиц.
Позволяет решить некоторые задачи, а также отвечает за создания шлейфа.
 

NiCollisionSwitch
 — нода полного отключения просчета любых столкновений.
Аналог NCO, но позволяющая делать много большее!
Например, можно активировать объекты находящиеся за видимым барьером!
Что невозможно получить никаким иным образом.
 

NiFltAnimationnode
 — нода автоматического переключения вложенных объектов.
Позволяет быстро создавать анимации из сменяющих друг друга шейпов (или нод с шейпами).
Близкий родственник FlippControllerа только для шейпов, а не для текстур.
 

NiNode
 — обычный тип контейнера не предполагающий особых настроек поведения.
Т.е. самый ходовой тип ноды, встречается во всех ниф файлах.
Даже нельзя назвать эту Ноду именной... но пускай будет))
 
NiLODNode
— нода управляет сменой уровней детализации геометрии объектов в зависимости от удаления камеры.
Level of Detail.
 
NiSortAdjustNode
— нода отвечает за улучшенную работу альфа канала между двумя объектами с прозрачностью.
Т.е. призвана препятствовать мерцанию таких объектов.
 
NiSwitchNode
— нода переключатель.
При использовании скриптов позволяет динамически переключать отображение содержимого.
Является близким "родственником" niLodNode, но управляется "руками", т.е. скриптами.
Активно использовалось в курице, например:
Растение с ягодами, переключается на модель без ягод посредством этой ноды.
В оригинальном МВ - сломано, или не активно.
Однако МСП 2.4 и МВСЕ 2.0 исправляют это безобразие.
Что позволило улучшить фпс и упростить замену объектов.
Требует МВСЕ 2.0 или выше!
 
RootCollisionNode
— нода используется для упрощения просчета столкновений.
Что, по идее, должно улучшать ФПС.
Также позволяет управлять зонам доступными для прохождения.
